# FinalSummary v1

## 用途

`FinalSummary` 是所有回合结束后的关卡最终总结页。它不是单回合结算页，也不是评分页；它用于把本次测试关卡中的回合历史、变量轨迹和学习要点集中展示出来。

## 五个模块

1. **回合历史**
   - 从 `GameState.round_history` 读取每一回合。
   - 展示已执行政策、结算方式和简要结果。

2. **宏观变量轨迹**
   - 从初始状态和每轮 `result.after` 生成文字轨迹。
   - 当前只显示文字，不绘制折线图。

3. **学习总结**
   - 使用固定模板和 `round_history` 中的机制文本汇总本关学习要点。
   - 不使用 AI 生成，不重新计算模型。

4. **评分系统占位**
   - 当前不计算评分，不显示分数，不做胜负判定。
   - 后续评分系统会根据关卡标签评价当前模型视野下的目标。

5. **长期视角提示**
   - 只在最终总结页显示。
   - 提醒玩家当前短期 IS-LM 关卡只评价短期需求管理效果。

## 数据来源

`FinalSummary` 只读取：

- `GameState.get_current_scenario()`
- `GameState.get_initial_state()`
- `GameState.round_history`

页面不调用 `MacroEngine`，不重新求解 IS-LM，也不推导新的政策结果。

## 评分占位原则

评分系统当前只是 UI 占位。后续实现时应与关卡标签绑定，例如：

- 短期｜价格刚性｜IS-LM：评价短期需求管理目标；
- 长期或综合关卡：再评价长期增长、财政可持续性等目标。

不属于当前时间视野的目标，只能作为提示，不计入当前关卡评分。

## 长期视角边界

长期视角提示只放在最终总结页，不放在每回合 `Result` 页面。当前不实现长期副作用计算，也不实现长期评分。

## 信息分工

- `PolicyDesk` 右侧信息面板：快速查看当前政策执行后状态。
- `Result` 本轮总结页：展示本轮 before -> after、机制总结、下一步操作。
- `IS-LM` 模型回放：展示曲线移动和 E0/E1。
- `FinalSummary` 最终总结页：展示全关卡回合历史、变量轨迹、学习总结、评分占位和长期视角提示。
